# macOS TCC blocks LaunchAgent access to ~/Desktop — use symlink

## What happened
The Discord bot LaunchAgent couldn't start because macOS TCC blocks processes spawned by launchd from reading files under `~/Desktop`, `~/Documents`, and `~/Downloads`.

## What was learned
Create a symlink from a non-TCC-protected path to the project:
```bash
ln -s ~/Desktop/AI-Harness ~/.local/ai-harness
```
Then point the LaunchAgent plist at `~/.local/ai-harness/...` instead of `~/Desktop/AI-Harness/...`. Symlinks are transparent to Node.js, Python, and git. No Full Disk Access needed.

## Why it matters
Granting Full Disk Access to `/opt/homebrew/bin/node` would let ANY Node.js process read all protected directories — a significant security risk from npm supply chain attacks. The symlink approach has zero security trade-offs.
