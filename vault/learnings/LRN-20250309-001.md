# Never spawn Claude CLI from Node.js child_process

## What happened
The Discord bot used Node.js `child_process.spawn()` to run Claude CLI. The process hung indefinitely — no output, no error, no timeout. Even using Python as an intermediary still stalled when stdout was piped back to Node.js.

## What was learned
Claude CLI is itself a Node.js/Bun app. Spawning it from another Node.js process causes pipe deadlocks. The fix requires **two things**:
1. Use Python `subprocess.run()` with a clean environment (no CLAUDE* env vars)
2. Write output to a **temp file** instead of stdout — Node.js pipes stall even with Python in between
3. Spawn Python with `detached: true, stdio: 'ignore'` and poll for the output file

## Why it matters
Without this fix, the Discord bot cannot communicate with Claude at all. This is the core functionality of AI Harness. The fix took multiple hours of debugging across PTY, script(1), and pipe-based approaches before landing on file-based output.
