# Claude CLI hangs when spawned from Node.js

## Command
`spawn("claude", ["-p", "--output-format", "json", "hello"])` from bot.ts

## Error
Process hangs indefinitely. No stdout, no stderr, no exit code. CPU at 0%.

## Environment
- macOS Sequoia (Darwin 24.6.0)
- Node.js 23.11.0
- Claude Code CLI (latest)
- Known bug: https://github.com/anthropics/claude-code/issues/771

## Root Cause
Claude CLI is a Node.js/Bun app that forks workers. When spawned from another Node.js process, the pipe handling between parent and child event loops deadlocks. Even Python as an intermediary stalls when its stdout is piped back to Node.js.

## Fix
File-based output approach:
1. Python `subprocess.run()` with `capture_output=True, stdin=DEVNULL`
2. Write result to a temp `.json` file (atomic: write to `.tmp` then `os.rename()`)
3. Node.js spawns Python with `detached: true, stdio: 'ignore'`
4. Node.js polls for the output file every 1 second
